Viva questions
Q: What is a network slice?
A logically isolated network customized for a specific service, running on top of shared physical infrastructure.
Q: What is S-NSSAI?
Single Network Slice Selection Assistance Information — composed of SST (slice type: 1=URLLC, 2=eMBB, 3=mMTC) and SD (differentiator). Used to identify slices across all 5G NFs.
Q: Can a slice exist without a dedicated UPF?
Yes — multiple slices can share a single UPF. Slice and UPF are different concepts. SMF decides which UPF serves which slice.
Q: Which metric degrades first in a congested shared UPF?
Delay — packets queue up first. Then jitter increases, followed by packet loss, and finally throughput collapse.
Q: Where must S-NSSAI be configured in Open5GS?
In 5 places: UE config (UERANSIM), subscriber profile (MongoDB), AMF, SMF, and UPF yamls. Mismatch in any one causes slice rejection.
⭐ A network slice is a logically isolated virtual network on shared physical infra, identified by S-NSSAI (SST+SD). Slices ≠ UPFs — SMF decides UPF mapping. Shared UPF causes resource contention (delay→jitter→loss→throughput). Your thesis measures exactly this degradation and asks when it becomes problematic.
