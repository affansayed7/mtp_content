Control plane vs user plane — in your lab
🧠 Control plane path
Used during: registration, auth, slicing
PC-1 → PC-2 → PC-3
AMF · AUSF · UDM · NSSF · SMF
Protocol: NGAP over SCTP
🛣️ User plane path
Used during: ping, iperf3, video
PC-1 → PC-2 → PC-3 (UPF only)
UPF only — no other NF
Protocol: GTP-U over UDP
Engineer mindset — 0 Mbps vs low Mbps are different problems
0 Mbps
Connectivity problem
Path is broken somewhere. Check routing, NAT, UPF running, GTP-U tunnel.
Low Mbps
Performance problem
Path works but congested. Check UPF CPU, queue build-up, shared UPF contention.
Never check CPU/bandwidth when throughput = 0 — fix the path first
Complete debugging cheatsheet
❌ Registration fails
→ UE config (IMSI, OPC, MCC/MNC)
→ gNB ↔ AMF connectivity (SCTP 38412)
→ AMF/AUSF/UDM/SCP running
→ MongoDB subscriber record
→ tail -f /var/log/open5gs/amf.log
❌ PDU session fails / no IP
→ SMF running
→ UPF running + ogstun interfaces exist
→ SMF ↔ UPF PFCP association
→ Slice config (SST/SD) in SMF + UPF yamls
→ tail -f /var/log/open5gs/smf.log
❌ IP assigned but ping fails
→ NAT/MASQUERADE rule exists?
→ iptables -t nat -L -n
→ IP routing on PC-3
→ ip route on UPF host
→ UPF GTP-U forwarding
⚠️ iperf3 low throughput
→ Shared UPF contention?
→ UPF CPU usage
→ Queue build-up / buffer overflow
→ NIC bandwidth saturation
→ Check delay + jitter too, not just throughput
Viva questions
Q: Registration OK, 10.45.0.2 assigned, iperf3 = 0 Mbps. First 3 checks?
1) Check NAT/MASQUERADE rules (iptables -t nat -L -n). 2) Check UPF routing (ip route on PC-3). 3) Check UPF is still running (systemctl status open5gs-upfd). CPU/bandwidth only after confirming traffic is actually flowing.
Q: Which components are in the data path during iperf3?
Only UPF (and gNB for radio). AMF, AUSF, UDM, NSSF, SMF are control-plane only — they don't touch data packets.
Q: Authentication traffic — does it cross the switch between PC-2 and PC-3?
No — AMF→AUSF→UDM→MongoDB is all internal to PC-3. Only the initial Registration Request (UE→gNB→AMF) and the challenge/response cross the network.
Q: What protocol carries user plane traffic between gNB and UPF?
GTP-U (GPRS Tunnelling Protocol - User Plane) over UDP. Control plane uses NGAP over SCTP between gNB and AMF.
⭐ Your lab: PC-1 (UEs) → M4300 Switch → PC-2 (gNB) → M4300 Switch → PC-3 (Full 5G Core). Control plane (auth, registration, slicing) uses NGAP/SCTP. User plane (iperf3, ping) uses GTP-U through UPF only. 0 Mbps = broken path. Low Mbps = congestion. Always distinguish these before debugging.
