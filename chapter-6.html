Chapter 6 — PDU session & internet access
5G Core fundamentals · quick reference
What is a PDU session?
Definition
A logical data connection between UE and a Data Network
After registration, UE is known to network but has no IP, no route, no internet. PDU session provides all three.
Who manages it?
SMF is the hero here
SMF decides:
· Which UPF to use
· Which IP to assign
· Which route to set
· Which slice it belongs to
Registration = "Who are you?"  ·  PDU Session = "Let's create your internet connection"

Actual traffic path (data plane)
UE → gNB → UPF → Internet
AMF disappears from traffic path.
UPF is the only user-plane component.
AMF handles registration. SMF handles session. UPF carries all actual traffic. Never mix these roles.
Why traffic bypasses AMF — the key design principle
🧠 Control plane — decisions
AMF — registration
SMF — session management
AUSF/UDM — auth
Never touches actual packets
🛣️ User plane — packets
UPF — forwards everything

YouTube, ping, iperf3, camera — all go through UPF only. 10 Gbps of traffic? AMF never sees a packet.
How UE traffic reaches internet — NAT
The problem
UE gets private IP 10.45.0.2 — not routable on public internet. Google doesn't know how to reply to 10.45.0.2.
NAT translation at UPF
SRC: 10.45.0.2
DST: 8.8.8.8
→
SRC: Host_Public_IP
DST: 8.8.8.8
UPF applies MASQUERADE / NAT rule
SRC: 8.8.8.8
DST: Host_Public_IP
→
SRC: 8.8.8.8
DST: 10.45.0.2
UPF translates reply back to UE
In Open5GS: iptables -t nat -A POSTROUTING -s 10.45.0.0/16 ! -o ogstun -j MASQUERADE

Debugger checklist — no internet after registration
1
Did UE get an IP?

2
Is UPF running?

3
Does NAT rule exist?

Is routing correct?

Viva questions
Q: What is a PDU session?
A logical data connection between UE and a Data Network, providing IP address and internet path via SMF + UPF.
Q: Which NF creates the PDU session?
SMF — selects UPF, assigns IP context, establishes the data path.
Q: Why does UE traffic not pass through AMF?
Control plane (AMF/SMF) makes decisions, user plane (UPF) carries packets. This separation prevents AMF from being overloaded by high-volume traffic.
Q: UE gets 10.45.0.2 but ping 8.8.8.8 fails. What do you check?
NAT/MASQUERADE rules at UPF — private IP 10.45.0.2 needs NAT to be routable on public internet. Also check UPF routing and ogstun interfaces.
Q: What is the difference between shared and dedicated UPF in slicing?
Shared UPF: all slices share one UPF instance. Dedicated UPF: each slice gets its own UPF for traffic isolation. Your thesis compares performance between these two.
⭐ After registration, SMF creates a PDU session — assigning IP (10.45.0.x) and selecting UPF. UPF carries all user traffic and applies NAT so private UE IPs can reach the public internet. IP assignment alone does not guarantee internet access — NAT rules must exist.
