Chapter 4 — 5G Core architecture
5G Core fundamentals · quick reference
Why not one giant server?
❌ Old approach
One monolithic server doing everything:
Auth + Mobility + Routing + Policies + Slicing

Hard to debug. Hard to scale. One crash = everything down.
✅ 5G approach — microservices
Each function does ONE job.
AMF → registration
AUSF → authentication
UPF → traffic forwarding

Easier to debug, scale, and maintain.
5G Core = Microservices architecture — each NF has exactly one responsibility
Most important concept — control plane vs user plane
🧠 Control Plane
AMF — registration, mobility
AUSF — authentication
UDM — subscriber data
SMF — session management
NSSF — slice selection
PCF — policies
NRF — service discovery
Makes decisions. Never touches user packets.
🚚 User Plane
UPF — packet forwarding

Carries actual traffic:
YouTube, ping, iperf3, camera streams

10 Gbps of video traffic? → UPF handles it. AMF never sees a single packet.
Memorize this split — it explains 80% of 5G architecture decisions
Network functions — quick reference
AMF
Access & Mobility Management Function
🛂 Reception desk
Registration, mobility, auth coordination.
✗ Does not carry traffic
✗ Does not assign IP
AUSF
Authentication Server Function
🔐 Security officer
Runs challenge-response. Verifies Ki.
UDM
Unified Data Management
📂 Records office
Owns subscriber data logically. MongoDB stores it physically.
SMF
Session Management Function
🎫 Flight booking
Creates PDU sessions. Picks UPF, assigns IP, sets route.
UPF
User Plane Function
🛣️ The highway
Forwards actual user packets. Most critical for your thesis. If UPF fails → no internet even if auth/registration succeed.
NRF
Network Repository Function
📞 Phone directory
Service discovery — tells AMF where SMF is, where AUSF is, etc.
NSSF
Network Slice Selection Function
🚦 Traffic allocator
Selects which slice a UE belongs to. Critical for your SDN slicing thesis.
PCF
Policy Control Function
📜 Rule book
Defines QoS, bandwidth policies, traffic rules. AMF and SMF consult PCF.
SCP
Service Communication Proxy
🔀 Router
Routes SBI messages between NFs indirectly. If SCP is down → all 504 errors.

Viva questions
Q: Which component carries user traffic?
UPF — User Plane Function. Only component in the user plane.
Q: Which component creates PDU sessions?
SMF — picks UPF, assigns IP, sets route.
Q: Which component selects slices?
NSSF — Network Slice Selection Function.
Q: What is the difference between control plane and user plane?
Control plane (AMF, SMF, AUSF etc.) makes decisions. User plane (UPF) forwards actual packets. They are separated so high-volume traffic doesn't overload control logic.
Q: What happens if NRF is down?
NFs can't discover each other — all SBI communication fails. Start NRF first, always.
⭐ 5G Core is a microservices architecture split into control plane (AMF, SMF, AUSF, UDM, NSSF, PCF, NRF) for decision-making, and user plane (UPF) for actual packet forwarding. This separation is the single most important design principle in 5G.
