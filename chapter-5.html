Chapter 5 — Registration flow end-to-end
5G Core fundamentals · quick reference
Registration — step by step
1
Cell search
UE powers on with no idea which network is nearby. Searches for any gNB broadcasting on correct PLMN (MCC+MNC). In lab: UERANSIM UE searches for UERANSIM gNB.
2
Radio connection (RRC)
UE establishes radio link with gNB. Physical link ✓ Auth ✗ Internet ✗
3
Registration Request → AMF
UE sends Registration Request via gNB → AMF. Contains IMSI/SUCI (identity). Think: student showing ID card at university gate.
4
Authentication — AMF → AUSF → UDM
AMF triggers auth via AUSF. UDM checks subscriber exists and returns Ki. AUSF runs challenge-response. If Ki mismatch → stops here.
5
Security mode setup
Network and UE establish encrypted security context. Like HTTPS handshake — protects all subsequent messages from eavesdropping.
6
Registration Accept + Complete
AMF sends Registration Accept. UE replies Registration Complete. UE is now officially registered. Registered ✓ Internet still not guaranteed ✗
Common failure points
❌ Failure at Step 3 — Request never reaches AMF
Wrong AMF IP in gNB config
AMF service down
Firewall blocking SCTP/38412
gNB ↔ AMF connectivity broken
❌ Failure at Step 4 — Authentication fails
Wrong IMSI (subscriber not in DB)
Ki / OPC mismatch
AUSF or UDM service down
SCP down → 504 errors
Know which step failed first — subscriber issues only matter AFTER the request reaches AMF

If gNB not found
Check PLMN (MCC/MNC), TAC, gNB config, RF settings
If AMF not reached
Check AMF IP in gNB config, AMF service status, firewall (SCTP 38412)
If auth fails
Check IMSI in DB, Ki/OPC values, AUSF/UDM running, SCP running
If registered but no internet
Check SMF, UPF, TUN interfaces, IP routing, NAT
Staged debugging model — most important mental model
0
UE can't find gNB
Check: PLMN config, TAC, gNB RF settings, UERANSIM gnb.yaml
1
UE found gNB, AMF not reached
Check: gNB ↔ AMF connectivity, AMF IP, open5gs-amfd status, SCTP port 38412
2
AMF reached, authentication fails
Check: IMSI in MongoDB, Ki/OPC match, AUSF/UDM/SCP running, amf.log for 504 errors
3
Registration OK, no internet
Check: SMF session creation, UPF running, ogstun interfaces, IP routing, NAT rules
Always identify the stage first — don't jump to subscriber issues if AMF never received the request
Viva questions
Q: Which component receives Registration Request?
AMF — via gNB over NGAP/N2 interface.
Q: What is the order of steps in registration?
Cell search → RRC connection → Registration Request → Authentication → Security setup → Registration Accept → Registration Complete.
Q: UE found gNB but AMF never receives request. What do you check?
gNB ↔ AMF connectivity — wrong AMF IP in gNB config, AMF service down, or firewall blocking SCTP port 38412. Subscriber issues are irrelevant at this stage.
Q: Can registration succeed without internet access?
Yes — registration and PDU session are separate phases. Internet needs PDU session via SMF + UPF after registration.
⭐ Registration follows: cell search → RRC → Registration Request (AMF) → authentication (AUSF+UDM) → security setup → Registration Accept. Debug by stage — identify exactly where the failure occurred before investigating the cause.
